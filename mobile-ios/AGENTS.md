# AGENTS.md — `mobile-ios`

Перед работой прочитай `../babangida-vault/COMMON.md`. Релевантные ADR: 0007, 0008.

## Роль

iOS-клиент: Dioxus + нативный шелл. Самая рискованная часть проекта — берись за
неё после стабильного веба.

## Границы

- Зависит на `domain`, `uikit`, `shared` (+ dioxus и нативный мост).
- `domain`/`shared` держим чистыми от Dioxus — чтобы план Б (нативка/RN) стоил только UI.
- Данные — через сеть/API, не через `infrastructure` напрямую.
