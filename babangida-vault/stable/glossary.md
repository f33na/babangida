# Глоссарий — ubiquitous language

Единый язык babangida: что значит термин в продукте и в коде. Описываем _термины_,
а не _типы_ — структуры (`Invite`, `Profile`, …) и инварианты живут в крейте
`domain` (ADR-0003). Здесь — смысл и границы понятия, одной строкой.

Формат записи: **термин** — краткое определение; при необходимости связь с ADR.
Наполняется по мере роста домена.

## Контекст identity-invites

- **invite** — приглашение, единственный путь регистрации. Правила выдачи — ADR-0005.
- **invite code** — код приглашения, по которому инвайт принимают. Генерится на границе, домен валидирует.
- **issuance (выдача)** — акт создания инвайта. Ограничен квотой и кулдауном (ADR-0005).
- **cooldown** — пауза между выдачами инвайтов (~12ч, ADR-0005). На админа не действует.
- **quota** — лимит одновременно активных (непринятых) инвайтов: 2; админ — без лимита (ADR-0005).
- **active invite** — выпущенный, ещё не принятый инвайт; только такие считаются в квоте.
- **handle** — уникальный @-идентификатор юзера (нормализуется в нижний регистр).
- **role** — роль юзера: **member** (рядовой) или **admin** (без лимита/кулдауна на инвайты).
- **verified** — статус верификации: открывает маркет, музыку, open API (ADR-0010).
- **casual** — (1) статус верификации: невериф. юзер (лента/профиль/переписка); (2) вариант субкультуры. Контекст различает.

## Контекст social

- **profile** — профиль юзера (display name, субкультура, био), один-к-одному с юзером.
- **display name** — отображаемое имя (не путать с `handle`).
- **subculture** — субкультура профиля: underground / casual / skin / hiphop / … (набор курируемый).
- **underground** — культурный контур комьюнити; задаёт тон продукта и модерации.
- **skin** — субкультура (вариант `subculture`); точный смысл закрепляется комьюнити.
- **public** — публичная сущность/паблик; значение уточняется в messaging-контексте (позже).

## Контекст content

- **post** — запись юзера в ленте.
- **feed** — лента постов; читается через read-модель CQRS (ADR-0004), не через агрегаты.
