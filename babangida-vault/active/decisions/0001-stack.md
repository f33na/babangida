# ADR-0001: Технологический стек

## Status

Accepted — 2026-06-25

## Context

Соцсеть с инвайт-доступом для underground RU hip-hop комьюнити, на старте — соло или маленькая команда. Нужен бэкенд, web и mobile (iOS/Android), при этом ресурсов на параллельную поддержку нескольких языков и экосистем нет. Главный ограничитель — пропускная способность одного-двух разработчиков, а не нагрузка.

## Options considered

Бэкенд:

- **Go.** Быстрый компайл, простой найм, зрелая экосистема. Минус — слабая система типов для domain-first DDD, нет нормальных алгебраических типов и исчерпывающего матчинга, моделирование инвариантов уезжает в рантайм-проверки.
- **Elixir/Phoenix.** Отличный для realtime и соцсетевых фич из коробки (LiveView, presence). Минус — динамическая типизация, маленький найм-пул в RU, и нулевой шанс переиспользовать код на фронте/мобиле.
- **TypeScript/Node.** Один язык фронт+бэк, огромный пул людей. Минус — слабые гарантии в рантайме, тяжёлый для долгоживущего домена, и всё равно не покрывает нативный mobile нормально.
- **Rust.** Сильная система типов под моделирование домена (enum/`Result`/newtype делают невалидные состояния непредставимыми), один язык на весь стек. Минус — маленький найм-пул, медленная компиляция, молодые web/mobile-фреймворки.

Full-stack: разные языки на фронт и бэк (например TS-фронт + Go-бэк) против единого Rust. Разные языки — это два набора инструментов, две модели данных на границе, дублирование типов и валидации. Единый Rust позволяет держать `shared` и `domain` крейты как единый источник правды и для бэка, и для web (Leptos), и для mobile (Dioxus).

## Decision

Полностью Rust-стек (Rust stable 1.95, edition 2024): Axum 0.8 + PostgreSQL 16 + sqlx 0.9 на бэке, Leptos 0.7 (SSR) на web, Dioxus 0.6 с нативным шеллом на mobile, Nix flake на окружение/CI/БД. Выбрали Rust ради сильной системы типов под domain-first архитектуру и возможности шарить доменный код между бэком, web и mobile одним языком — для команды из одного-двух человек это важнее, чем простота найма. Точные версии — живой список в `babangida-vault/stable/stack.md`.

## Consequences

Получаем: один язык на весь стек, общие `shared`/`domain` крейты, компилятор как первая линия защиты инвариантов домена, единое воспроизводимое окружение через Nix.

Теряем и усложняем: найм-пул по Rust маленький, особенно в RU underground-контексте — расширяться будет тяжело. Время компиляции бьёт по скорости итераций, особенно холодные сборки. Leptos и Dioxus молодые и быстро меняются: придётся жить с breaking changes и местами писать то, что в зрелых экосистемах берётся готовым. Ставка на то, что выигрыш от единого типобезопасного стека перевесит эти издержки на горизонте проекта.
